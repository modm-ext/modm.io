    Supported Targets¶

    modm supports a total of 3831 targets by setting the modm:target repository option with the device identifier string.

    This is the long string written on your microcontroller that uniquely identifies the target device for which to generate the modm HAL.

    You can optionally add the hardware revision to the identifier string by appending /revX where X is the revision. This can help modm apply workarounds for specific hardware bugs as recommended by the errata sheet.

    Hosted OS¶

    The hosted targets are for compiling modm on an operating system, usually to run our unit tests locally or in the CI.

    Identifier format example for Hosted-Linux:

      hosted    linux
{platform}-{family}

    hosted-darwin, hosted-darwin-arm64, hosted-linux, hosted-linux-arm64, hosted-windows.

    Microchip AVR¶

    Identifier format example for ATmega328P-AU: 

        avr     mega   328    p              au
{platform}{family}{name}{type}-{speed}{package}
    Note that the platform is mapped to avr not at and that some AVRs have an optional device speed.

    Identifier format example for AT90CAN128-16AU: 

        avr      90    can   128     16      au
{platform}{family}{type}{name}-{speed}{package}
    Note that the type and name are switched here to normalize the identifier schema.
